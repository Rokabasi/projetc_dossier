Les structures des données de dépannage d'un protocole OSPF à zones multiples

OSPF stocke les informations de routage dans quatre structures de données clés :

  • Table d'interfaces : cette table dresse la liste de toutes les interfaces qui ont été activées pour OSPF. Les annonces LSA (Link-state advertisement) de type 1 contiennent les sous-réseaux associés à chaque interface active.

  • Table de voisinage : cette table permet de gérer les contiguïtés de voisinage via les intervalles d'attente Hello et Dead. Les entrées de voisinage sont ajoutées ou mises à jour lors de la réception d'un message « hello ». Les voisins sont supprimés à l'expiration du minuteur « dead ».

  • Base de données à état de liens (LSDB) : c'est la structure de données principale utilisée par OSPF pour stocker des informations sur la topologie réseau. Elle inclut des informations topologiques complètes sur chaque zone à laquelle le routeur OSPF est connecté et sur tous les chemins disponibles pour atteindre d'autres réseaux ou systèmes autonomes.

  • Table de routage : après avoir calculé l'algorithme SPF, les meilleures routes vers chaque réseau sont proposées à la table de routage.
