La vérification des rôles du routeur désigné (DR)/du routeur désigné de secours (BDR)

Dans la topologie à accès multiple illustrée à la Figure 1, trois routeurs sont interconnectés par un réseau à accès multiple Ethernet commun, 192.168.1.0/28. Chaque routeur est configuré avec l'adresse IPv4 indiquée sur l'interface Gigabit Ethernet 0/0.

Comme les routeurs sont connectés sur un réseau commun à accès multiple avec diffusion, OSPF a sélectionné automatiquement un DR et un BDR. Dans cet exemple, R3 a été sélectionné comme DR car son ID de routeur est 3.3.3.3, soit le plus élevé de ce réseau. R2 est le BDR car son ID de routeur vient en deuxième position dans ce réseau.

Pour vérifier les rôles du routeur OSPFv2, utilisez la commande show ip ospf interface (Figure 2). Le résultat produit par R1 confirme les aspects suivants :

1. R1 n'est ni le DR, ni le BDR ; c'est un DROther, avec une priorité par défaut égale à 1. (1)

2. Le DR est R3, avec l'ID 3.3.3.3, à l'adresse IPv4 192.168.1.3, tandis que le BDR est R2, avec l'ID 2.2.2.2, à l'adresse IPv4 192.168.1.2. (2)

3. R1 comporte deux contiguïtés, l'une avec le BDR et l'autre avec le DR. (3)

Le résultat généré par R2 dans la figure 3 confirme que :

1. R2 est le BDR avec une priorité par défaut de 1. (1)

2. Le DR est R3, avec l'ID 3.3.3.3, à l'adresse IPv4 192.168.1.3, tandis que le BDR est R2, avec l'ID 2.2.2.2, à l'adresse IPv4 192.168.1.2. (2)

3. R2 comporte deux contiguïtés, l'une avec un voisin dont l'ID de routeur est 1.1.1.1 (R1) et l'autre avec le DR. (3)

Le résultat généré par R3 dans la figure 4 confirme que :

1. R3 est le DR, avec une priorité par défaut égale à 1. (1)

2. Le DR est R3, avec l'ID 3.3.3.3, à l'adresse IPv4 192.168.1.3, tandis que le BDR est R2, avec l'ID 2.2.2.2, à l'adresse IPv4 192.168.1.2. (2)

3. R3 comporte deux contiguïtés, l'une avec un voisin dont l'ID de routeur est 1.1.1.1 (R1) et l'autre avec le BDR. (3)

Remarque : pour obtenir la commande OSPFv3 équivalente, remplacez ip par ipv6.

Dans la figure 1, il y a trois routeurs connectés à un commutateur. Le routeur 2 est le routeur DR et le routeur 3 est le routeur BDR. La Figure 2 présente le résultat de la commande « show i. p. o. s. p. f. interface gigabit ethernet zero slash zero » sur le routeur 1. Le routeur 1 est un routeur DROther. La Figure 3 présente le résultat de la commande « show i. p. o. s. p. f. interface gigabit ethernet zero slash zero » sur le routeur 2. Le routeur 2 est le routeur BDR. La Figure 4 présente le résultat de la commande « show i. p. o. s. p. f. interface gigabit ethernet zero slash zero » sur le routeur 3. Le routeur 3 est le routeur DR.
